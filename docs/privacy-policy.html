1. Scope

EazyFill provides user-controlled assistance for configured CAPTCHA fields, reusable form-filling rules, and user-managed scripts. Users choose the sites, selectors, rules, profiles, settings, scripts, and actions involved. EazyFill does not promise unattended or universal automation.

This policy covers information processed by the EazyFill browser extension and its supporting services. A payment provider or other third party may have its own privacy terms when you interact directly with that provider.

2. Information EazyFill Handles

Registration and account information

Registration can process personally identifiable information (PII), including an email address or mobile number. An optional account name, account, plan, subscription, credit, quota, and account-verification status can also be processed.

Authentication and device information

EazyFill processes API keys and key status. It generates a random installation/device identifier and can process a device name, user-agent metadata, first-seen and last-seen timestamps, and device status. The service can associate the API key and device information for authenticated access, device limits, support, and abuse prevention.

User-created content and settings

EazyFill processes user-created form-filling rules, userscripts and userscript storage, profiles, CAPTCHA selectors, selected element selectors, captured form steps, settings, credit state, and sync metadata. This information reflects the workflows and page actions that a user chooses to configure.

Selected website content and limited web activity

When a configured feature is invoked or enabled, EazyFill can process the current website domain, selected page elements, selected CAPTCHA images or text, image dimensions, and related request metadata. EazyFill does not need to build a general browsing-history list, but it does process this limited domain and website-content information to perform requested features and operate security and usage controls.

Usage, diagnostic, security, and network information

Operational records can include task type, result or operation status, processing time, model identifier, solve and credit counts, quota state, domain, account or key association, IP address, timestamps, errors, reliability data, and other network or security metadata. These records can identify activity associated with an account, API key, or device.

Billing and provider records

Billing records can include user/account and plan identifiers, amount, currency, payment method or provider, order identifiers, payment identifiers, references, status, provider status, signed verification data, webhook payloads, and timestamps. Payment instruments and checkout credentials, such as card, UPI, or wallet details, are entered into and handled by the selected payment provider rather than by the extension.

3. How Information Is Used

EazyFill uses the information described in this policy to:

  • Register and administer accounts, API keys, devices, plans, subscriptions, credits, and quotas.
  • Authenticate service requests and enforce access and device limits.
  • Perform user-requested CAPTCHA assistance and configured form or script actions.
  • Store and apply the rules, scripts, profiles, selectors, and settings a user creates.
  • Provide optional encrypted synchronization when a user enables and invokes it.
  • Process purchases, verify payment events, and maintain billing and provider records.
  • Provide support, diagnose failures, measure reliability, secure the service, and prevent abuse.
  • Meet legal obligations and enforce applicable terms.

4. Local Browser Storage

The extension stores settings, the API-key record, rules, scripts, profiles, CAPTCHA selectors, credit state, and sync metadata in browser extension storage. API keys, rules, scripts, profiles, and supported userscript storage records use EazyFill's protected local storage layer. Not every local setting or metadata field is encrypted.

Locally stored content is used to perform the actions a user configures. Website changes, unsupported frames, browser restrictions, or invalid selectors can prevent an action from working.

5. Optional AES-GCM Encrypted Sync

Cloud sync is optional and disabled by default. When a user invokes sync, EazyFill creates a payload containing form-filling rules, userscripts, profiles, CAPTCHA selectors, settings, and sync-version metadata. An apiKey field is excluded from the sync payload.

The extension encrypts the payload before upload using AES-GCM with key material derived from the API key and generated device identifier. The service stores the encrypted blob and operational metadata, including the device identifier, version, blob hash, blob size, and timestamps. The service implementation does not receive the plaintext sync payload. Losing or changing the required API-key or device context can prevent decryption.

Users can request deletion of the cloud sync blob through the extension. The production timing for deletion from active systems and backups is TBD before publication. No immediate or fixed-period deletion promise is made in this draft.

6. Requested CAPTCHA Processing

For configured CAPTCHA assistance, EazyFill can read the user-selected source element and target field. A solve request can send a selected CAPTCHA image encoded for transport or selected text, the current website domain, image dimensions or related request metadata, and the API key and generated device identifier in request headers.

The CAPTCHA payload can be processed by the API, an in-process or remote queue, solver workers, and a duplicate-result cache keyed from a hash of the request. Related operational records can contain task type, status, processing time, model identifier, domain, account or key association, IP address, and timestamps.

Production retention periods for CAPTCHA payloads, queues, worker data, caches, logs, backups, and infrastructure records are TBD before publication.

7. Billing and Payments

EazyFill can create orders, verify payment events, process provider notifications, update plan or credit state, and retain the billing records described above. Credits or quotas can be reserved for a requested operation, and the service can attempt to restore them when processing fails before a result is returned.

Available plans, allowances, payment methods, and providers depend on the production configuration. Production payment-provider names are TBD before publication. A provider's own privacy terms apply when a user enters payment details into its checkout.

8. Sharing, Service Providers, and No Sale

EazyFill does not sell user data. EazyFill does not use user data to build advertising profiles or for targeted advertising, and it does not use or transfer user data to determine creditworthiness or for lending.

Information can be disclosed only as needed to provide and secure EazyFill, comply with law, protect users or the service, or complete a transaction. Relevant provider categories can include:

  • Cloud hosting, storage, database, networking, queue, cache, and backup providers.
  • CAPTCHA processing and solver infrastructure providers.
  • Account, authentication, communications, customer-support, and operational service providers.
  • Security, abuse-prevention, monitoring, logging, and error-diagnostic providers.
  • Payment, checkout, bank-transfer, and billing providers.
  • Professional advisers, regulators, courts, law enforcement, or other parties when legally required or necessary to protect rights and safety.

The exact provider categories used must be checked against the final deployment. Production provider names are TBD before publication.

9. Data Retention and Deletion

Local extension data remains in browser storage until it is removed through an available EazyFill control, cleared through browser controls, or removed with the extension, subject to browser behavior. Uninstalling the extension does not by itself delete records already held by the supporting service or a payment provider.

Server-side account, device, sync, CAPTCHA, usage, security, network, support, and billing records are retained as needed for feature delivery, account administration, security, abuse prevention, diagnostics, transaction records, dispute handling, legal compliance, and backups. Different record types can have different retention periods.

Exact production retention periods, deletion-processing times, and backup expiration schedules are TBD before publication. Deletion requests may require verification and may not remove records that must be kept for legal, security, fraud-prevention, accounting, or dispute-resolution purposes.

10. Your Choices and Controls

Depending on the released feature set, users can:

  • Choose which sites, selectors, rules, profiles, scripts, and actions to configure.
  • Disable individual features, rules, scripts, or CAPTCHA assistance.
  • Leave optional cloud sync disabled or enable and invoke it.
  • Request deletion of the encrypted cloud sync blob through the extension.
  • Remove available local rules, scripts, profiles, selectors, settings, or other extension data.
  • Clear browser extension data or uninstall EazyFill.
  • Revoke or replace an API key and manage available device or account controls.
  • Contact EazyFill about an access, correction, deletion, or privacy request once the public contact is published.

Some choices can limit or prevent the affected feature from working. Legal rights concerning personal information vary by location.

11. Security

EazyFill uses technical and organizational measures designed to protect information, including encrypted HTTPS service communication, protected local storage for specified records, authenticated API requests, and client-side AES-GCM encryption for optional sync. Not every local field is encrypted, and no method of storage or transmission is completely secure.

12. Children

EazyFill is not directed to children under 13, and EazyFill does not knowingly collect personal information from children under 13. Any additional age restrictions required for the production service must be confirmed before publication.
